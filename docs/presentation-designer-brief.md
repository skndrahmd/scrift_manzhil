## DESIGN GUIDELINES (Global)

**Color Palette:**

| Swatch | Name | Hex | Usage |
|---|---|---|---|
| 🟢 | Manzhil Dark | `#075E54` | Primary brand, headings, dark backgrounds |
| 🟢 | Manzhil Teal | `#128C7E` | Secondary brand, subheadings, accents |
| 🟢 | Manzhil Light | `#25D366` | Bright accent, success states, CTA buttons |
| 🟢 | Manzhil Success | `#34B27B` | Confirmation highlights |
| ⚪ | White | `#FFFFFF` | Slide backgrounds |
| ⚪ | Subtle Gray | `#F9FAFB` | Alternating table rows, subtle backgrounds |
| ⚪ | Highlight Teal | `#F0FDF9` | Light teal tint for chat bubbles |
| ⬛ | Text Primary | `#111827` | Main headings |
| ⬛ | Text Secondary | `#374151` | Body text |
| ⬛ | Text Tertiary | `#6B7280` | Captions, labels |
| ⬛ | Border Light | `#E5E7EB` | Dividers, table borders |

**Typography:** DM Sans

**General Principles:**
- Professional teals and whites. No flashy colors beyond the palette.
- Every slide answers: "What's in it for the building manager?"
- Tone: Confident but approachable
- NO technical jargon (no "API", "database", "webhook", "server")
- WhatsApp chat mockups should look like real WhatsApp conversations (green bubbles for Manzhil, white for user)
- Use phone frame mockups (iPhone/Android) around WhatsApp conversations

### "Managing a Building Shouldn't Be This Hard"

Every building manager knows the struggle:

- **Endless phone calls** — Residents calling at all hours for complaints, updates, payment questions
- **Paper-based tracking** — Complaints written on scraps of paper, things falling through the cracks

**Sound familiar? There's a better way.**
